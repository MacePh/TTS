Core Concepts

Before we can dive any further, we must introduce some core concepts of React and some of the nifty things that create-react-app has done making the “weird” coding style we just saw possible.

 

JSX

Most React code is written in a superset of JavaScript called JSX. It lets you create components in your JS code in a way that looks more familiar (it uses HTML-like syntax). Keep in mind, this IS NOT HTML (no matter how much it looks like it). It's also not really JS. It needs to be compiled by babel (through Webpack) into a .js file that a browser can read. This is all already handled by create-react-app. 

Intro to React

Intro to React

Components

Components let you split the UI into independent, reusable pieces, and think about each piece in isolation.

// Defining a component using a Javascript function "Functional Component"
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}
 
 
// Defining a component using an ES6 class "Class Component"
class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

Components

Everything in React is a component. Think of a component as a custom html tag that renders its own interface and comes packaged with its own behavior (click events, etc.) which the programmer defines.

React components can only return one element so everything must be wrapped in a single element (usually a <div>), and have two main ways of keeping track of information: props and state.

Virtual DOM
React creates a JS representation of the DOM, this is called the “Virtual DOM”. React renders a virtual representation of the DOM in memory (invisible to us) in which it holds the Components mentioned above, and manages when the data changes inside a Component (prop or state), this gives programmers a much simpler programming model and an easier interface to the DOM.

Intro to React

ReactDOM

The library we'll be using to render React components to an HTML document is called reactDOM. ReactDOM converts your React state to HTML. It manages the HTML and event handlers for you so that subsequent renderings of your application make the fewest possible changes to the actual DOM, increasing performance.

Important - React abstracts away the DOM interface. You're not really writing HTML.

Intro to React

JSX Caveats

Like we said before, JSX isn't really HTML.

Because it lives in the same context as JavaScript, it has some special rules you need to follow when returning components from your render method.

  • Attributes with the same names as reserved JS keywords can't be used. For example, class is a reserved keyword in both JS and HTML, so in JSX we use className in reference to the HTML class.
  • All tags have to be closed. <input type="text">, for example, has to include a closing tag <input type="text" />.
  • For the most part, however, JSX and HTML line up pretty well. Keep your eye out though, as some areas are tricky!

Intro to React

The Problem

We know from creating three notes that we are reusing the Note component, and the only thing we really want to change is the text inside it. Right now, in order to do this,  we'd have to create and hardcode 3 different variations of the Note component.  

And honestly, we can't even reuse the current Note component we have (if we change the text in it, it will change for all Notes, right?).

Intro to React

The Solution

  • We need to pass the initial text from our Board to our Note
  • How do we pass data from a parent component (Board) down to a child component (Note)?

There are two ways: State and Props

Although either could solve our problem, it is best practice to only use props (think short for property) when passing data from a parent component to a child. We use state when a component needs to keep track of its own data. Another good rule of thumb is that props should be used with data that the component isn’t going to change, and use state with data that will (e.g. a notes content).

Don’t worry if we don’t understand the difference yet, once we use them in our application it will become clearer.

Token Replacement

In order to populate our component with values from the props, we will need to use something called token replacement, which is a key part to any templating language, like JSX.

 

Token replacement is simply taking variables and putting their value into the template.

In JSX, use {} (curly braces) with the desired value. Essentially these curly brackets allow us to run JavaScript within our JSX.

<h5 className="card-title">{this.props.title}</h5>

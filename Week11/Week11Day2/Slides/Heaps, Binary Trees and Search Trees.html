Heaps, Binary Trees, and Search Trees

What Are Trees?

  • Node - Also known as the key
  • Edge - An edge connects two nodes to show that there is a relationship between them. Every node except the root is connected by one incoming edge from another node. Each node may have several outgoing edges.
  • Root - The root of the tree is the originating node. It has no incoming edges.
  • Path - A path is an ordered list of nodes that are connected by edges. 

Terminology

class Node{
	int key; //data held in Node
	Node leftNode; //left child node
	Node rightNode; //right child node
}
 
class Tree{
	Node root; //notice that to start a tree, we only actually need the root node. 
}
  • Children - The nodes with incoming edges from the same node are the children of that node. 
  • Parent - A node is the parent of all the nodes it connects to with outgoing edges. 
  • Sibling - Nodes in the tree that are children of the same parent are siblings.

Terminology cont...

  • Subtree - A subtree is a set of nodes and edges comprised of a parent and all the descendants.
  • Leaf Node - A leaf node indicates the end of one path along the tree as it is a node that has no children.

Terminology cont...

  • Every node has a path of length 0 to itself
  • Every node has only one path to it from the root
    • depth: length of the path from the root to the node
    • height: length of the longest path from the node to a leaf
    • height of tree: is the height of the root
  • So, a root has depth 0 and a leaf has height 0

Tree Terminology

Binary Trees

 

A Binary Tree works a lot like a LinkedList, but each of its nodes holds a reference to not one, but two other objects.

 

Each node can have a left child and a right child, and each of those children can have children.

Binary Tree Structure Property

Full: A full Binary Tree is a tree in which every node except for the leaves has exactly two children. 

 

Complete: A complete binary tree refers to a tree in which each level (except the last possibly) is full and all the nodes are as far to the left as possible. 

Tree Traversals

We visit all nodes in some order, following edges in two ways:

1. Depth first

Basic idea is to travel down paths towards leaves until forced to stop

 

2. Breadth first

Basic idea is to visit all children of a node before going deeper down paths

Depth First Traversals

1. PreOrder

Visit root, then recursively visit children (L to R)

2. PostOrder

Visit children (L to R) and finally visit root

3. InOrder

Visit L child, then visit root, then visit R child

 

For n-ary tree, we have to decide where in the “middle” to put root visit

A node-based binary tree data structure has the following properties:

  • The left subtree of a node only contains nodes with keys less than the node’s key.
  • The right subtree of a node only contains nodes with keys greater than the node’s key.
  • The left and right subtree each must also be a binary search tree.

What happened?

Total nodes in a complete binary tree = 2^(height+1)-1

 Conversely, height of a complete binary tree = floor(log2 n)

1) Start at the root.

2) If the search key is less than the index key of the element, go to the left child. If the search key is greater than the index key of the element, go to the right child.

 

This process continues until the element is found, or until the end of the tree is found (in which case the element does not exist). In three comparisons, we have either found the element, or it does not exist.

Implementing a BST

A recursive method for inserting a Node in the tree: 

public class BinaryTree {
	Node root; 
	
	public Node insertNode(Node current, int key) {
		
		if(current == null) {
			return new Node(key);
		}else if(key < current.getKey()) {
			current.setLeftNode(insertNode(current.getLeftNode(), key));
		}else if(key > current.getKey()) {
			current.setRightNode(insertNode(current.getRightNode(), key));
		}else {
			return current; 
		}
		
		return current; 
	}
    
    public void add(int key) {
		root = insertNode(root, key); 
	}

Heaps

  • Heaps are just specialized Binary Trees, with two additional characteristics/rules.
  • Because of these rules, you use a heap only when you care about the highest or lowest value in a set of numbers.

Heaps

  1. The tree must be complete and balanced (structure property)
    • Must be filled out top-to-bottom, left-to-right
  2. The tree must follow certain ordering rules. (order property)

There are two kinds of heaps:

Min Heaps

  • The parent value <= its children
  • The smallest value will always be at the top

Max Heaps

  • The parent value >= its children
  • The largest value will always be at the top


Heap-structure property

Complete Tree

Not Complete

Not Complete

A complete binary tree is where every level, except possibly the last, is filled.

Heap-order property (min-heap)

  • Every path from root to leaf is an ordered sequence small to large
  • Min at root
  • Every child >= parent

Heap-order property (min-heap)

  • Every path from root to leaf is an ordered sequence large to small
  • Max at root
  • Every child <= parent

Binary Heap Implementation

Structure property allows very efficient representation with an array

  • Leave slot 0 unused
  • Store node val as an array element
  • For a node in slot i
    • The parent is at  i/2 (floor, int division)
    • Left child is  2*i
    • Right child is (2*i)+1

 

Binary Heap Implementation

  • Fill the array, breadth-first across the tree, placing the root at index 1.
  • Each level is then placed in the subsequent index locations (Level 1 -> index 2-3)... 

Tree From Array

  • Parent: floor(1/2) is 0 (root, no parent)
  • L-Child: 2*1 is 2, slot 2 has 7
  • R-Child: (2*1)+1 is 3, slot 3 has the value 4

public void Minadd(int value) {
	if(heap.size()>1) {
		int parentIndex = Math.floorDiv(heap.size(), 2); 
		int parentValue = heap.get(parentIndex);
		if(value >= parentValue) {
			heap.add(value);
		}else {
        	heap.add(parentValue);
			if(parentIndex == 1) {
				heap.set(1, value);
			}else {
				int newParentIndex = Math.floorDiv(parentIndex, 2);
				int newParentValue = heap.get(newParentIndex);
				do {
					if(newParentIndex == 1) {
						break; 
					}
					heap.set(parentIndex, newParentValue);
					parentIndex = newParentIndex; 
					parentValue = newParentValue; 
					newParentIndex = Math.floorDiv(parentIndex, 2);
					newParentValue = heap.get(newParentIndex);
				} while(value < newParentValue);
				heap.set(parentIndex, newParentValue);
				heap.set(newParentIndex, value);
			}
		}	
		}else {
			heap.add(value);
		}
	}
	
public void Maxadd(int value) {
	if(heap.size()>1) {
		int parentIndex = Math.floorDiv(heap.size(), 2); 
		int parentValue = heap.get(parentIndex);
		if(value <= parentValue) {
			heap.add(value);
		}else {
				heap.add(parentValue);
		if(parentIndex == 1) {
				heap.set(1, value);
			}else {
				int newParentIndex = Math.floorDiv(parentIndex, 2);
				int newParentValue = heap.get(newParentIndex);
				do {
					if(newParentIndex == 1) {
						break; 
					}
					heap.set(parentIndex, newParentValue);
					parentIndex = newParentIndex; 
					parentValue = newParentValue; 
					newParentIndex = Math.floorDiv(parentIndex, 2);
					newParentValue = heap.get(newParentIndex);
				} while(value > newParentValue);
				heap.set(parentIndex, newParentValue);
				heap.set(newParentIndex, value);				
            }
		}	
	}else {
		heap.add(value);
	}
}

Binary Heap Deletion Operation

Deleting a leaf has no effect on the tree and is therefore

straight-forward

 

Deleting a node with 1 child involves simply replacing the value in the node to be deleted with the value in the child node. 

 

However, deleting a node that has 2 or more children can be more complicated as you must shift values in the tree to not break the linkages between nodes. 

Binary Heap Deletion Operation

Deletion of an internal node with two children is a bit tricky. If we delete a node like this, we split a tree into two subtrees, and some children of the internal node won't be accessible after deletion.

What To Do

Replace the node being deleted with the largest node in the left subtree and then delete that largest node.

 

 

 

 

 

 

By symmetry, the node being deleted can be swapped with the smallest node in the right subtree.

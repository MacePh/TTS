What is Java?

  • A programming language developed by Sun Microsystems in 1995

  • Remains the most widely used programming language due to its flexibility in when and where it can be used

  • Slogan is "Write Once, Run Everywhere", promising to provide the flexibility of developing and running Java code on any device or platform

  • In November of 2006, Sun began releasing Java's core code as free, open-source software

  • In 2009, Oracle acquired Sun and now maintains the language

Java vs. JavaScript

Java and JavaScript, while sounding similar, are not the same or related. Sort of like ham and hamburger, park and parkway, can and canary… you get the idea!

Benefits of Java

In Java, everything is considered an Object.

This allows the code to be easily extended with the use of classes and methods and allows you to write less code to perform common tasks.


Java follows the concepts of Object Oriented Programming (OOP) and is designed to be easy to learn and use

Object Oriented

Architecture and Platform Independent

Java compiles into a platform-independent bytecode and interpreted by the Java Virtual Machine (JVM).

 

Multithreaded

This feature allows for the development of interactive applications that can perform many tasks simultaneously, while still running smoothly.

Benefits of Java

While Java is a compiled programming language, it only compiles down to an intermediate binary form called JVM (Java Virtual Machine) bytecode and is not yet executable machine code.

 

The language compiler (javac) compiles the language into bytecode which is understood by the JVM.

 

The bytecode is interpreted by the JVM and converted to native machine code. 

Compiled and Interpreted

JDK (Java Development Kit), the JVM (Java Virtual Machine), and the JRE (Java Runtime Environment) provide the necessary components for writing and running Java applications on your machine. 

 

Practically speaking, when you install the JDK (when you install java) you will also get the JVM and JRE.

By convention only one variable is declared at a time, although it is syntactly legal to declare multiple variables on a single line.

The reasoning behind the convention is that:

all variables must be initialized before they can be used. There will be a compile error if a variable is used before it has been assigned a value.

// legal, but runs the risk of forgetting to initialize (assign values)
int pop, top, can; 
 
// legal but not recommended
int pop=0, top=0, can=0;
 
// recommended
int pop = 0;
int top = 0;
int can = 0;

The String Type

The Java type String is a class type provided by the Java language. It is not a native type but is often thought of as fundamental since it is not user-defined and is provided by the framework itself.  

Strings are created through either literal assignment or by using the String constructor.

String s1 = "Hello";
String s2 = new String("World");

The String Type

At its core, the String type is a list (collection) of primitive characters. The String class gives us a convenient way of handling and working with what is, essentially, an array of chars. 

One of the fundamental things to do with a collection is to measure how many items are in it.

int len = s1.length();  // equals 5
bool sameLen = (s1.length() == s2.length()); // equals true

Strings can also be combined to build bigger Strings.

String greeting = s1.concat(s2);  // greeting will now hold "HelloWorld" with no space.
String longGreeting = s1.concat(" ").concat(s2).concat(" from Java!");  
// longGreeting will now be "Hello World from Java!"

The String Type

Thankfully, the standard addition operator is overloaded to also work on Strings.

String longGreeting = s1 + " " + s2 + " from Java!";
// longGreeting will now be "Hello World from Java!"

int ndex = longGreeting.indexOf('W');             // ndex equal to 6

String word2 = "Hello";
String word3 = "Goodbye";

word1.equals(word2); //true
word1.equals(word3); //false

// These two have the same value
new String("Hello World").equals("Hello World") // --> true 

// "==" checks if they are the same object, and they are not the same object
//One is a string object the other is a literal
new String("Hello World") == "Hello World" // --> false 

// These are both String objects, but not the same 
new String("Hello World") == new String("Hello World") // --> false 

// Literals are interned strings and seen as the same object by the compiler 
"test" == "test" // --> true 

// String literals are concatenated by the compiler yet the results are not seen as the same object.
"Hello World" == "Hello " + "World" // --> false

When checking to see if two string are equal, we use Object.equals() method.
